Future Watermasters working code Briefs should consider the recon pass as routine pre-work rather than optional.

- *Provenance-based Operator output identity for stable operators.* Brief 3's key decision: an Operator's output Artifact id derives from `(operation, inputs, params)`, not from output bytes. This is because `stable` operators produce byte-different but semantically-equivalent output across runs; byte-based identity would fork artifact identity for runs that should be the same. Provenance-based identity makes "the same operator on the same inputs with the same params" yield the same artifact id regardless of byte variation. The pattern composes correctly with `determinism-class.md` v1: for `stochastic` operators, the seed is already in params, so provenance derivation includes it automatically.

- *Strategy-tagged hashing for cross-strategy collision prevention.* When multiple derivation strategies coexist for the same identity-bearing field (`Artifact.id` in Brief 3, but the pattern generalizes), prefix each strategy's hash input with a distinct tag (`b"src:"`, `b"prov:"`, `b"ref:"`) so two strategies cannot collide on the same hex output even with adversarial inputs. Small pattern, worth keeping in mind for future hash-based identity work.

- *Engineer scope-discipline track-record matters.* Codex's scope discipline held across all three Briefs — every write inside `write_scope`, every deviation named openly in the chat return. The discipline shaped my Brief structure over time: by Brief 3 I included an explicit "If you see useful adjacent work, flag it in the return rather than writing it" constraint, which had concrete grounding from Brief 1's experience. The Engineer pattern's track record across multiple Briefs in a session can inform the design of subsequent Briefs.
